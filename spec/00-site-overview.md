# Сайт приложения Eagle Security — обзор

Сайт посвящён одному продукту — приложению **Eagle Security** (Android, анти-абзор, IMSI-catcher, защита камеры/микрофона). Репозиторий: [site-eaglesecurity](https://github.com/nedimonvam/site-eaglesecurity).

Это не визитка компании, а **продуктовый лендинг**: контент и набор страниц задаются отдельным ТЗ (см. [01-tz-template.md](01-tz-template.md)).

---

## Общие требования к сайту

### Внешний вид и стиль

- **Минималистичный** дизайн: текст, при необходимости — логотип приложения и скриншоты/иконки продукта.
- **Моноширинные шрифты (mono)** для основного текста и заголовков, опционально — комбинация с одним акцентным шрифтом по ТЗ.
- **Цветовая схема**: тёмный фон, светлый текст; допустимы акценты (например зелёный/терминальный стиль или палитра из ТЗ). Контраст — не ниже WCAG AA.
- **Без тяжёлой анимации**; при необходимости — лёгкие эффекты (подчёркивание, hover).

### Языки

- **Два языка**: немецкий (DE) и английский (EN).
- Переключение языка на каждой странице (например в футере: DE | EN).
- Контент и навигация редактируются отдельно для каждого языка (Markdown/данные по локалям).

### Обязательные разделы по закону (Германия, ИП)

Сайт обязан содержать (см. DDG / Impressumspflicht):

1. **Impressum** — страница с полными данными поставщика:
   - имя/название ИП с правовой формой,
   - полный почтовый адрес,
   - e-mail,
   - быстрый способ связи (телефон или контактная форма),
   - при наличии: Handelsregisternummer, USt-IdNr.,
   - ответственный за контент.

2. **Datenschutz / Privacy Policy** — страница с политикой конфиденциальности:
   - кто ответственный,
   - какие данные обрабатываются (логи, cookie, формы и т.д.),
   - цели и правовые основания,
   - сроки хранения,
   - права пользователей и контакт для запросов.

Ссылки на Impressum и Datenschutz должны быть легко доступны на каждой странице (как правило — в футере).

### Техника и деплой

- **Редактирование контента**: по возможности — простое (Markdown, YAML/JSON); точный способ задаётся в ТЗ.
- **Сборка**: статический сайт (SSG или ручная вёрстка), без серверной логики на продакшене.
- **Деплой**: один скрипт/команда для публикации (например push в ветку `gh-pages` или выгрузка статики на хостинг).
- **Локальный просмотр**: один скрипт/команда для запуска сайта на localhost (например Eleventy `--serve` или простой HTTP-сервер).

### Хостинг

- Размещение на **простом и дешёвом** хостинге: GitHub Pages, Netlify, Cloudflare Pages или статика по FTP. Домен и базовый URL задаются в ТЗ.
